## v1.1.1 更新说明

本版本新增**最小审计集**和**维度29因果链检查**，解决v1.1.0中"28维度太复杂，AI跳过执行"的问题。

| 问题 | v1.1.0 | v1.1.1 |

## 严重程度模型

- **critical** — 破坏核心逻辑/连续性/主角锁定/平台红线
- **major** — 明显削弱章节或损害铺垫/回报
- **minor** — 风格、清晰度、重复、局部节奏
- **note** — 可选改进或观察

## 最小审计集（v1.1.1核心）

每章写完后**必须执行**以下5项检查，不可跳过：

### 检查1：字数检查

```
目标：2000-2500字（可根据book_rules.md调整）
检查：统计中文字符数（wc -c / 3）
结果：
  ✅ 通过：2000 ≤ 字数 ≤ 2500
  ⚠️ 不足：字数 < 2000 → 必须扩充
  ⚠️ 超标：字数 > 3000 → 建议拆分
```

### 检查2：AI去味检查（智能替换）

```
步骤：
  1. 搜索禁用词（不禁、竟然、居然、仿佛、宛如、深邃、命运、瞳孔收缩、嘴角上扬）
  2. 分析语境（是否在对话中？是否必要？）
  3. 智能替换（见ai-detox-guide.md替换表）
  4. 语义完整性验证
  5. 记录替换结果
```

### 检查3：因果链检查（维度29，v1.1.1新增）

```
检查项：
  1. 每个场景转换是否有因果连接？
  2. 是否有"突然""然后"连接无因果的事件？
  3. 主角行为是否有动机支撑？
  4. 新角色出场是否有前置铺垫？

检查流程：
  for each 场景转换:
      if 转换无因果连接:
          → 报警：场景跳脱
          → 要求：添加过渡句或因果事件
  
  for each "突然"/"然后":
      if 前后无因果关系:
          → 报警：虚假连接词
          → 要求：补充因果或删除连接词
```

**典型问题**：
- 主角在茶馆喝茶，下一段突然出现在废弃工厂（无过渡）
- "突然，一个黑影出现在他身后"（无前因）

### 检查4：行为-规则一致性检查（维度28）

```
检查项：
  1. 本章是否声明了规则？
  2. 规则说X，主角是否做了非X？
  3. 如果做了非X，是否有明确解释？
  4. 其他角色行为是否符合规则？

检查流程：
  for each 规则声明:
      规则内容 = 提取规则(声明)
      for each 后续行为:
          if 行为违反规则:
              if 无解释:
                  → 报警：行为-规则矛盾
                  → 要求：添加解释或修改行为
```

**典型问题**：
- 规则说"灯闪时不要移动"，主角直接走出（无解释）

### 检查5：时间线检查（v1.1.1新增）

```
检查项：
  1. 时间跳跃是否交代了中间发生了什么？
  2. 是否存在"时间黑洞"（跳跃>1小时无交代）？

检查流程：
  for each 时间跳跃:
      跳跃时长 = 结束时间 - 开始时间
      if 跳跃时长 > 1小时:
          if 无中间事件交代:
              → 报警：时间黑洞
              → 要求：补充中间事件摘要或时间标记
```

**典型问题**：
- 主角11点到小区，凌晨3点才进楼（中间2小时无事件）

## 一、核心维度（完整审计）

> 以下维度建议每10章执行一次完整检查

### 1. 角色一致性（OOC）

检查命名角色是否与其已建立的特质、能力、恐惧、骄傲、智力和最近情感状态一致。

**问题**：
- 角色是否在没有铺垫的情况下变得被动/仁慈/懦弱？
- 对话是否突然改变语体？
- 主角是否忘记了长期的行为锁定？

### 2. 时间线连续性

检查顺序、持续时间、旅行时间、冷却时间、伤害和恢复窗口是否合理。

**问题**：
- 事件是否发生得太快？
- 时间是否意外改变？
- 是否忽略了前一章的约束？

### 3. 世界观一致性

检查魔法系统、技术、政治、地理、机构、等级、经济、禁忌规则和因果关系。

**问题**：
- 章节是否违反了硬性世界规则？
- 地点或机构是否在没有解释的情况下突然表现不同？

### 4. 信息边界泄漏

检查每个角色被允许知道什么。

**问题**：
- 角色是否引用了只有其他人知道的信息？
- 叙述是否隐含地剧透了POV知识之外的内容？
- 外传是否泄漏了未来正典信息？

### 5. 资源/物品/伤害追踪

检查金钱、物品、武器、线索、伤害、体力、修炼资源、社会资本和义务。

**问题**：
- 丢失的物品是否重新出现？
- 严重的伤害是否消失？
- 能力输出是否忽略了已建立的成本？

### 6. 钩子管理

检查对读者做出的承诺。

**问题**：
- 钩子是否意外丢失？
- 回报是否仓促或不劳而获？
- 是否打开了太多钩子而没有进展？

### 7. 情节逻辑

检查动机、决策、场景因果关系和后果流动。

**问题**：
- 行动是否因果相关？
- 是否有人拿着"傻瓜球"？
- 冲突是否通过便利而非铺垫解决？

### 8. 节奏与章节功能

检查章节是否有明确的工作和足够的进展。

**问题**：
- 章节是否推进了情节、关系、谜团或情感弧线？
- 是否在拖延？
- 是否过度解释而不是戏剧化？

### 9. 情感弧线一致性

检查最近章节的情感延续。

**问题**：
- 恐惧/愤怒/悲伤是否在场景之间消失？
- 情感升级或释放是否感觉理所当然？

### 10. 对话真实性

检查角色是否听起来独特且符合情境。

**问题**：
- 对话是否过于说明性？
- 不同角色是否听起来一样？
- 台词是否太整洁或太AI干净？

## 二、风格与反公式维度

### 11. 重复/疲劳词

检查重复的短语、隐喻、句子模式和情感标签。

**番茄AI味高频词**：
- "不禁"、"竟然"、"居然"
- "仿佛"、"宛如"
- "深邃"、"冰冷"
- "命运"、"时光"、"岁月"
- "瞳孔收缩"、"嘴角上扬"

### 12. 报告式语言/抽象解释

标记听起来像分析、总结、励志演讲或写作建议而不是生活叙事的散文。

**典型表现**：
- "命运的齿轮开始转动"
- "时间仿佛静止了"
- 突然开始讲人生道理

### 13. 过度通用的群体反应

标记陈词滥调，如全场震惊、普遍沉默、所有人倒吸一口凉气等，除非特别赢得。

### 14. 讲述而非展示

标记可以用一个具体动作更好地工作的总结行。

**示例**：
- "他感到愤怒" → "他把杯子摔在地上"
- "她很伤心" → "她转过身，肩膀在抖"

### 15. 段落单调

检查等长段落、重复的过渡模式和列表式节奏。

### 16. AI式过渡密度

检查过度使用"突然""瞬间""仿佛""在那一刻""不禁""意外地"等词。

---

## 三、结构维度

### 17. 大纲偏移

检查章节是否仍然服务于预期的弧线。

### 18. 支线停滞

检查侧线是否冻结太久。

### 19. 承诺/回报不匹配

检查读者期望是否管理良好。

### 20. 场景进出纪律

检查场景是否开始太早、结束太晚或没有节拍变化地漂移。

---

## 四、衍生作品维度

### 21. 父级正典冲突

对于前传、续集、外传或同人，检查章节是否与父级正典矛盾。

### 22. 未来信息泄漏

检查角色是否知道只应在正典时间线后期出现的揭示。

### 23. 钩子隔离

检查衍生作品是否不当解决了属于父级作品的钩子。

---

## 五、番茄专用维度

### 24. 平台红线检查

检查是否触犯番茄平台红线。

**检查项**：
- [ ] 是否标注"AI辅助创作"？
- [ ] 是否有敏感内容？
- [ ] 是否有抄袭/洗稿嫌疑？
- [ ] 是否有重复注水？

### 25. 生死线检查

前三章必须检查。

**检查项**：
- [ ] 前300字是否有主角+核心事件？
- [ ] 前1000字是否有身份+金手指+冲突？
- [ ] 第一章末是否有钩子？
- [ ] 前三章是否有核心能力+目标+悬念？

### 26. AI去味检查

每章必须检查（见最小审计集检查2）。

**检查项**：
- [ ] 是否有排比三连？
- [ ] 是否有空洞抒情？
- [ ] 是否有万能过渡句？
- [ ] 是否有万能形容词？
- [ ] 是否有情感标签？

### 27. 商业性检查

检查签约和付费转化。

**检查项**：
- [ ] 本章是否推进了剧情？
- [ ] 章末是否有钩子？
- [ ] 信息密度是否足够？
- [ ] 是否有爽点释放？

### 28. 行为-规则一致性审查（规则怪谈类必查）

见最小审计集检查4。

### 29. 因果链检查（v1.1.1新增）

见最小审计集检查3。

## 审计输出格式（v1.1.1简化）

```markdown
# 第X章审计报告

## 最小审计集（5项强制）

| 检查项 | 结果 | 详情 |
|--------|------|------|
| 字数检查 | ✅/⚠️/❌ | 实际字数：{{words}} |
| AI去味检查 | ✅/❌ | 发现{{count}}处，已替换为... |
| 因果链检查 | ✅/❌ | {{causal_detail}} |
| 行为-规则一致性 | ✅/❌ | {{rule_detail}} |
| 时间线检查 | ✅/❌ | {{time_detail}} |

## 综合评级
- 评级：✅通过 / ⚠️需小改 / ❌需大改

## 必改项（critical/major）
- ...

## 建议改项（minor/note）
- ...
```

|---------|------|------|
| 最小审计集 | 每章强制 | 5项 |
