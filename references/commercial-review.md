# 商业审查指南

> 适用：番茄小说平台，长篇网文
> 核心目标：通过签约审核 → 提升完读率 → 最大化收益

---

## 一、商业审查三角色

### 角色 A：番茄编辑（商业把关）

**身份**：番茄小说签约编辑，每天审50+投稿，只看前三章决定是否签约

**核心关注**：这本书能不能签约？能不能赚钱？

**检查清单**：
- [ ] 前300字是否出现主角+核心事件？
- [ ] 第一章结尾是否有钩子？
- [ ] 前三章是否完成"黄金三角"？
- [ ] 书名是否有吸引力？
- [ ] 题材是否在番茄热门赛道？
- [ ] 人设是否3秒可记住？
- [ ] 是否有明确的"爽点引擎"？

### 角色 B：资深读者（体验把关）

**身份**：番茄平台日均阅读3小时的老书虫，追过200+本网文

**核心关注**：这本书好不好看？我会不会弃书？

**检查清单**：
- [ ] 是否有"AI味"让我出戏？
- [ ] 对话是否自然？
- [ ] 人物是否有"人味"？
- [ ] 是否有"水章节"？
- [ ] 信息密度是否够？
- [ ] 主角是否让我有代入感？
- [ ] 反派是否有压迫感？
- [ ] 是否有"爽点"释放？

### 角色 C：逻辑质检员（质量把关）

**身份**：细节控，专门找bug

**核心关注**：这本书有没有硬伤？会不会被读者骂"降智"？

**检查清单**：
- [ ] 人物行为动机是否合理？
- [ ] 前后设定是否一致？
- [ ] 时间线是否合理？
- [ ] 因果关系是否成立？
- [ ] 金手指/系统是否有合理限制？
- [ ] 世界观规则是否自洽？
- [ ] 人物能力是否匹配其身份/经历？

## 二、黄金三章审查

### 第一章审查

| 检查项 | 要求 | 是否达标 |
|--------|------|---------|
| 前300字 | 出现主角+核心事件 | 是/否 |
| 前1000字 | 身份清晰+金手指+小冲突 | 是/否 |
| 完整冲突 | 开始→发展→解决/反转 | 是/否 |
| "卧槽"时刻 | 主角做了让读者意外的事 | 是/否 |
| 章末钩子 | 读者想看第二章 | 是/否 |

### 第二章审查

| 检查项 | 要求 | 是否达标 |
|--------|------|---------|
| 接第一章钩子 | 快速进入状态 | 是/否 |
| 推进主线 | 有明确进展 | 是/否 |
| 情绪高潮 | 反击/觉醒/获得 | 是/否 |
| 章末钩子 | 新的悬念 | 是/否 |

### 第三章审查

| 检查项 | 要求 | 是否达标 |
|--------|------|---------|
| 核心能力展示 | 读者知道主角凭什么赢 | 是/否 |
| 主角目标清晰 | 读者知道主角要干什么 | 是/否 |
| 大悬念抛出 | 读者愿意追更 | 是/否 |
| 爽点释放 | 至少一个爽场景 | 是/否 |

## 三、爽点引擎审查

### 3.1 爽的本质

**爽 = 期待 → 压抑 → 释放**

没有压抑就没有爽。主角一路平推不叫爽，叫无聊。

### 3.2 爽点的五种类型

| 类型 | 结构 | 频率 | 示例 |
|------|------|------|------|
| **反转爽** | 读者以为A，结果是B | 每1-2章 | 以为要死，反杀 |
| **碾压爽** | 主角用实力碾压对手 | 每3-5章 | 拖出镜中诡异踩住 |
| **智斗爽** | 主角用智商取胜 | 每2-3章 | 利用规则漏洞 |
| **情感爽** | 情感爆发（哭、笑、怒） | 每5-10章 | 师父牺牲 |
| **升级爽** | 获得新能力/新道具 | 每卷2-3次 | 规则碎片→规则改写 |

### 3.3 爽点必须有代价

- 用了规则屏蔽器 → 报废了
- 赢了战斗 → 受伤了
- 获得信息 → 暴露了

**没有代价的爽 = 没有张力 = 读者会腻**

### 3.4 爽点节奏曲线

```
一章内的理想节奏：

平静 → 小冲突 → 压抑 → 反转/爆发 → 余韵/钩子
 10%    30%      20%     30%        10%
```

## 四、悬念管理审查

### 4.1 悬念的三种类型

| 类型 | 定义 | 生命周期 | 示例 |
|------|------|---------|------|
| **小悬念** | 一个问题，1-3章内回答 | 1-3章 | 扫地人是谁？ |
| **中悬念** | 一个谜团，一卷内揭示 | 10-30章 | 父亲去哪了？ |
| **大悬念** | 终极问题，全书揭示 | 全书 | 沈默是什么？ |

### 4.2 悬念的黄金法则

- **永远不要一次性揭示悬念**：揭示一个的同时，抛出新的
- **揭示悬念 = 制造新的悬念**：知道父亲去了规则世界 → 他在规则世界里做了什么？
- **读者猜到的不要急着确认**：让读者猜3-5章，再给答案
- **不要解释悬念**：能通过行动/对话暗示的，不要直接说出来

### 4.3 章末钩子的四种写法

1. **新信息出现**：收到一条神秘短信
2. **反转**：以为安全了，其实没有
3. **选择**：主角面临两难选择
4. **悬念加深**：发现之前的认知是错的

## 五、付费转化审查

### 5.1 免费章节（前10-15章）要求

| 章节 | 要求 |
|------|------|
| 第1章 | 抓人开篇，建立期待 |
| 第2章 | 情绪高潮，展示能力 |
| 第3章 | 大悬念，确立目标 |
| 第4-10章 | 持续爽点，每章有钩子 |
| 第11-15章 | 付费前高潮，大悬念升级 |

### 5.2 付费节点要求

- [ ] 是否有足够强的悬念/冲突？
- [ ] 读者是否愿意付费看下一章？
- [ ] 是否有"不付费就难受"的感觉？

### 5.3 断章钩子检查

每章结尾检查：
- [ ] 是否有钩子？
- [ ] 钩子是否足够强？
- [ ] 读者是否想看下一章？

## 六、签约关键节点审查

### 6.1 第一次签约机会（2万字）

| 检查项 | 要求 |
|--------|------|
| 开篇质量 | 黄金三章通过 |
| 爽点引擎 | 明确且可持续 |
| 钩子密度 | 每1-2章有钩子 |
| 更新稳定性 | 日更4000字 |
| AI味 | 已去除 |

### 6.2 第二次签约机会（5万字）

| 检查项 | 要求 |
|--------|------|
| 剧情发展 | 有新的进展 |
| 追更欲望 | 读者愿意继续看 |
| 质量稳定 | 无明显下滑 |

### 6.3 第三次签约机会（8万字）

| 检查项 | 要求 |
|--------|------|
| 如仍未签约 | 需大改或切书 |
| 问题诊断 | 找出未签约原因 |
| 修改方案 | 针对性修改 |

## 七、数据追踪建议

### 7.1 关键数据

| 数据 | 追踪方式 | 调整方向 |
|------|---------|---------|
| 书架比 | 番茄后台 | 前3章不够抓人 → 修改开篇 |
| 追更比 | 番茄后台 | 钩子不够 → 加强章末悬念 |
| 完读率 | 番茄后台 | 中间拖沓 → 删减注水 |
| 评论区 | 看热评 | 读者喜欢什么/讨厌什么 |

### 7.2 数据健康指标

| 指标 | 健康值 | 警戒值 |
|------|--------|--------|
| 书架比 | >15% | <10% |
| 追更比 | >30% | <20% |
| 完读率 | >50% | <30% |
