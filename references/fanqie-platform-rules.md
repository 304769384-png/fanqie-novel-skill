# 番茄小说平台规则

> 适用对象：在番茄小说平台用AI辅助写作的新人作者
> 核心目标：通过签约审核 → 稳定日更 → 提升完读率 → 赚到钱

---

## 一、平台红线（必须优先检查）

> ⚠️ 以下任何一条触碰，作品直接不合格，无需继续评审

| 红线 | 说明 | 检查方法 |
|------|------|---------|
| **纯AI生成** | 2026年起纯AI作品永久封禁，必须有人工底稿 | 检查是否有"AI味"明显的段落（排比三连、空洞抒情、万能过渡句） |
| **未标注AI辅助** | 国家要求标注AI生成内容，未标注会被限流 | 确认作品已正确标注AI辅助创作 |
| **敏感内容** | 涉政、涉黄、涉暴、涉迷信等 | 检查是否有平台违禁词和擦边内容 |
| **抄袭/洗稿** | 包括抄袭其他小说或大量复制公开文本 | 检查是否有大段与已知作品高度相似的段落 |
| **重复注水** | 同一内容换措辞反复写、大段无意义描写 | 检查信息密度，删除不推进剧情的段落 |

## 二、生死线（前三章决定一切）

### 2.1 番茄读者行为数据

- **前300字**：决定读者是否继续看。30%的读者在这里流失。
- **前1000字**：决定读者是否加入书架。又流失30%。
- **前3000字（第一章）**：决定读者是否看第二章。又流失20%。
- **前三章**：决定读者是否付费。留存下来的只有20%。
- **前十章**：决定读者是否追更。最终留存约10-15%。

**结论：你写的每一句话，都在被读者审判。没有"后面会变好"这回事。**

### 2.2 前300字必须做到

- [ ] 第一句话就要有画面感或悬念
- [ ] 300字内必须出现异常/冲突/悬念，不能是日常描写
- [ ] 读者读完300字必须想问"然后呢？"

### 2.3 前1000字必须做到

- [ ] 主角身份清晰（谁？干什么？）
- [ ] 金手指/特殊能力出现（读者要知道主角凭什么赢）
- [ ] 第一个小冲突发生（不能全是铺垫）

### 2.4 第一章必须做到

- [ ] 至少一个完整的小冲突（开始→发展→解决/反转）
- [ ] 至少一个"卧槽"时刻（主角做了让读者意外的事）
- [ ] 章末必须有钩子（读者必须想看第二章）

### 2.5 前三章必须做到

- [ ] 主角的核心能力展示完毕
- [ ] 主角的目标清晰（他要干什么？）
- [ ] 至少一个值得追的悬念（父亲是谁？第七条规则是什么？）
- [ ] 至少一个让读者爽的场景

## 三、签约适配要求

### 3.1 题材适配

| 热门赛道 | 说明 | 适合长篇？ |
|---------|------|-----------|
| 规则怪谈+民俗悬疑 | 番茄2026最热赛道 | ⭐⭐⭐⭐⭐ |
| 低武仙侠+文明碰撞 | 经典长篇赛道 | ⭐⭐⭐⭐⭐ |
| 系统流职业文 | 都市热门 | ⭐⭐⭐⭐ |
| 反套路苟道/脑洞玄幻 | 高潜力赛道 | ⭐⭐⭐⭐ |

### 3.2 人设要求

- [ ] 人设是否3秒可记住？（身份标签 + 核心冲突 + 爽点引擎）
- [ ] 是否有明确的"爽点引擎"？（读者追更的核心动力是什么）
- [ ] 更新节奏是否适配？（日更4000字，每章2000字左右）

### 3.3 付费转化要求

- [ ] 免费章节（前10-15章）是否埋够钩子？
- [ ] 付费节点是否有足够强的悬念/冲突？
- [ ] 每章结尾是否有"断章钩子"？（让读者不得不看下一章）

## 四、新人常见踩坑

### 4.1 签约阶段踩坑

| 坑 | 后果 | 避坑方法 |
|----|------|---------|
| 前三章铺垫太多，第5章才进入主线 | 编辑直接拒签，读者划走 | 前300字出主角+事件 |
| 书名太文艺/太普通 | 点击率低，算法不推流 | 书名要包含核心卖点/冲突 |
| 简介写成了剧情梗概 | 没有吸引力，读者不点进来 | 简介要藏爽点和钩子 |
| 第一次签约机会（2万字）没过就放弃 | 浪费后续两次机会 | 每次机会都认真打磨再提交 |
| 同一设备多开账号 | 触发风控，限流或封号 | 一个设备一个账号 |

### 4.2 日更阶段踩坑

| 坑 | 后果 | 避坑方法 |
|----|------|---------|
| 断更/不稳定更新 | 算法降权，读者流失 | 提前存稿，保持日更 |
| 首秀期改正文/换封面 | 触发重新审核，流量归零 | 首秀期三不：不改正文、不换封面、不改简介 |
| 注水凑字数 | 完读率下降，算法降权 | 每章都要有信息量 |
| AI味太重被举报 | 限流或下架 | 每章必做去味检查 |
| 忽视后台数据 | 不知道哪里出了问题 | 每天看书架比、追更比、完读率 |

### 4.3 变现阶段踩坑

| 坑 | 后果 | 避坑方法 |
|----|------|---------|
| 免费章节钩子不够 | 读者不付费，收入低 | 前10-15章每章都要有钩子 |
| 付费后质量下降 | 读者退款、弃书 | 付费章节质量只能更高 |
| 不看评论区反馈 | 不知道读者真正想看什么 | 定期看评论，调整方向 |

## 五、日更节奏建议

### 5.1 日更4000字的时间分配

| 环节 | 时间 | 说明 |
|------|------|------|
| 构思本章大纲 | 15分钟 | 确定本章要推进什么、结尾钩子是什么 |
| AI辅助写作 | 60-90分钟 | 用AI生成初稿，边写边改 |
| 人工修改+去味 | 30分钟 | 重点去AI味、调整对话、优化节奏 |
| 评审检查 | 10分钟 | 按本规范快速评审 |
| **总计** | **约2-2.5小时** | 分两次写，上午2000字，下午2000字 |

### 5.2 存稿策略

- 开书前存稿**7-10章**（约2万字），确保第一次签约机会时质量稳定
- 保持**3-5章**的存稿缓冲，防止突发情况断更
- 每周末多写1-2章作为应急存稿

## 六、签约关键节点

| 节点 | 字数 | 检查重点 | 预期结果 |
|------|------|---------|---------|
| 黄金三章 | 前3章 | 开篇抓人、人设清晰、无硬伤、无AI味 | 读者愿意继续看 |
| 第一次签约机会 | 2万字 | 爽点引擎明确、钩子充足、质量稳定 | 通过签约审核 |
| 第二次签约机会 | 5万字 | 剧情有新发展、追更欲望强 | 如未签约，继续打磨 |
| 第三次签约机会 | 8万字 | 如仍未签约，需大改或切书 | 最后机会 |
| 全勤奖门槛 | 10万字 | 质量稳定、无重大硬伤 | 可拿全勤奖 |
