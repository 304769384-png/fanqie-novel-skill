# AI去味专项检查指南

> 适用：番茄小说平台，AI辅助写作
> 核心目标：避免AI味被平台限流，确保作品通过人工审核

---

## v1.1.1 更新说明

本版本新增**智能替换机制**，解决v1.1.0中"机械删除导致语义断裂"的问题。

| 问题 | v1.1.0 | v1.1.1 |
|------|--------|--------|
| 处理方式 | 搜索→删除 | 搜索→分析→替换→验证 |
| 语义检查 | 无 | 替换后必须语义完整 |
| 保留例外 | 无 | 对话中可保留 |

---

## 一、为什么AI味是致命问题

### 1.1 平台政策

- 2026年起，番茄平台对纯AI作品永久封禁
- AI味明显的作品会被限流或下架
- 读者举报AI味作品，会触发人工审核

### 1.2 读者体验

- AI味会让读者"出戏"，破坏沉浸感
- AI味通常表现为：空洞、重复、说教、万能过渡
- 读者能敏锐感知"这不是人写的"

## 二、智能替换表（v1.1.1核心）

### 2.1 禁用词替换规则

| 禁用词 | 替换方案 | 保留例外 | 示例 |
|--------|---------|---------|------|
| 仿佛 | 像是/如同/好比 | 对话中可保留 | "仿佛看见"→"像是看见" |
| 竟然 | （补充前文原因） | 表达角色惊讶时可保留 | "竟然还在"→"还在"（补充前文原因） |
| 居然 | 同"竟然" | 同上 | 同上 |
| 不禁 | （删除，补充动作） | 无例外 | "不禁笑了"→"笑了" |
| 宛如 | 如同/好似 | 对话中可保留 | "宛如仙境"→"如同仙境" |
| 深邃 | 深沉/幽深/漆黑/浓重 | 描写眼睛时用"漆黑" | "深邃的眼眸"→"漆黑的眼眸" |
| 命运 | 走向/轨迹/结局 | 对话中可保留 | "命运安排"→"际遇" |
| 瞳孔收缩 | （用具体反应替代） | 无例外 | →"眨了眨眼/后退半步/攥紧拳头" |
| 嘴角上扬 | （用具体动作替代） | 无例外 | →"笑了/露出笑意" |

### 2.2 替换流程

```
步骤1：搜索禁用词
  grep -n "不禁|竟然|居然|仿佛|宛如|深邃|命运|瞳孔收缩|嘴角上扬" chapter.md

步骤2：分析语境
  for each 匹配:
    - 是否在对话中？（引号内）
      → 是：可保留
      → 否：继续分析
    - 是否是必要的比喻？
      → 是：保留比喻，替换为更好的表达
      → 否：替换为具体动作/感官

步骤3：智能替换
  按替换表执行替换

步骤4：语义完整性验证
  - 替换后朗读测试
  - 确认没有产生歧义或断裂
  - 确认句子仍然通顺

步骤5：记录
  - 记录替换了哪些词、替换成了什么
  - 用于后续质量追踪
```

### 2.3 反面教材（v1.1.1重点）

**案例1：机械删除导致语义断裂**

```
原文：那双眼睛仿佛深不见底的黑潭。
机械删除：那双眼睛深不见底的黑潭。（❌ 语义断裂！）
智能替换：那双眼睛像深不见底的黑潭。（✅ 保留比喻，换词）
```

**案例2：删除后产生歧义**

```
原文：有人仿佛拿铁签捅他的眼睛。
机械删除：有人拿铁签捅他的眼睛。（❌ 产生歧义！像真的在捅！）
智能替换：有人像是拿铁签捅他的眼睛。（✅ 保留比喻）
```

**案例3：对话中可保留**

```
原文："这竟然是真的！"他惊呼。
错误修改："这是真的！"他惊呼。（❌ 失去惊讶语气！）
正确处理：保留，因为是对话。（✅）
```

## 三、高频AI味清单

### 3.1 排比三连

**典型表现**：
```
他看到了远处的山峦。
他听到了风的声音。
他感受到了命运的召唤。
```

**问题**：结构重复，像在填模板

**修改方向**：改为具体的单一感官描写
```
修改后：
远处的山峦在暮色中变成一道剪影，风从山谷吹来，带着松针的苦味。
```

---

### 3.2 空洞抒情

**典型表现**：
- "命运的齿轮开始转动"
- "时间仿佛静止了"
- "岁月如梭，光阴似箭"
- "命运的安排总是出人意料"

**问题**：没有实际内容，像在凑字数

**修改方向**：删除或替换为具体动作/对话
```
修改前：命运的齿轮开始转动，他不知道接下来会发生什么。
修改后：他攥紧了手里的纸条，盯着那行字看了三秒。
```

---

### 3.3 万能过渡句

**典型表现**：
- "然而，事情并没有那么简单"
- "就在这时"
- "就在此时此刻"
- "就在他准备离开的时候"

**问题**：用烂了，读者看到就知道"又要转折了"

**修改方向**：用事件本身推动过渡
```
修改前：然而，事情并没有那么简单。门突然开了。
修改后：门开了。
```

---

### 3.4 过度解释

**典型表现**：
```
陆沉很聪明，所以他立刻意识到了不对劲。他决定去查看一下，因为他觉得这件事很重要。
```

**问题**：读者能猜到的事不需要解释

**修改方向**：信任读者，删减解释
```
修改前：陆沉很聪明，所以他立刻意识到了不对劲。
修改后：陆沉停下了脚步。
```

---

### 3.5 完美对话

**典型表现**：
- 每个人说话都像在演讲
- 没有口癖、语气差异
- 对话过于书面化、说教味重

**问题**：不像真人在说话

**修改方向**：加入口语化、停顿、打断、口头禅
```
修改前："我认为我们应该仔细分析一下当前的情况，然后做出最合理的决定。"
修改后："我觉得……"他顿了一下，"算了，先看看再说。"
```

---

### 3.6 说教旁白

**典型表现**：
```
人生就是这样，你永远不知道下一秒会发生什么。有时候，命运会给你一个措手不及的惊喜。
```

**问题**：突然开始讲人生道理，破坏叙事

**修改方向**：删除或融入人物对话中
```
修改前：人生就是这样，你永远不知道下一秒会发生什么。
修改后：（删除，或通过人物对话表达）
"谁知道呢，"他耸了耸肩，"人生嘛。"
```

---

### 3.7 情感标签

**典型表现**：
- "他感到愤怒"
- "她很伤心"
- "他不禁感到恐惧"
- "她感到一阵温暖"

**问题**：直接告诉读者情感，而不是让读者感受

**修改方向**：用行为展现情感
```
修改前：他感到愤怒。
修改后：他把杯子摔在地上。

修改前：她很伤心。
修改后：她转过身，肩膀在抖。

修改前：他感到恐惧。
修改后：他的手在发抖。
```

## 四、AI去味自检流程

### 4.1 关键词搜索

每章写完后，全文搜索以下关键词，逐个检查是否必要：

**高频AI味词**：
- "不禁"、"竟然"、"居然"
- "仿佛"、"宛如"、"好像"
- "然而"、"但是"、"不过"（是否过多）
- "深邃"、"冰冷"、"温暖"（是否空洞）
- "命运"、"时光"、"岁月"（是否抒情过度）
- "瞳孔收缩"、"嘴角上扬"
- "就在这时"、"就在此时"

### 4.2 对话检查

随机抽3段对话，遮住说话人名字，能否区分是谁在说？

**检查项**：
- [ ] 不同角色说话方式是否不同？
- [ ] 是否有口癖/语气差异？
- [ ] 是否过于书面化？

### 4.3 朗读测试

把章节通读一遍，有没有读起来别扭/不自然的句子？

**检查项**：
- [ ] 是否有读起来拗口的句子？
- [ ] 是否有重复的句式？
- [ ] 是否有空洞的段落？

## 五、AI味检查表

### 5.1 每章必查

| 检查项 | 是否存在 | 修改建议 |
|--------|---------|---------|
| 排比三连 | 是/否 | 改为单一感官描写 |
| 空洞抒情 | 是/否 | 删除或替换为动作 |
| 万能过渡 | 是/否 | 用事件推动过渡 |
| 过度解释 | 是/否 | 删减解释 |
| 完美对话 | 是/否 | 加入口语化 |
| 万能形容词 | 是/否 | 用动作替代 |
| 说教旁白 | 是/否 | 删除或融入对话 |
| 情感标签 | 是/否 | 用行为展现 |

### 5.2 AI味密度检查

| AI味出现频率 | 风险等级 | 建议 |
|-------------|---------|------|
| 每章0-2处 | ✅ 低风险 | 可发布 |
| 每章3-5处 | ⚠️ 中风险 | 建议修改后发布 |
| 每章5处以上 | ❌ 高风险 | 必须大改 |

## 六、去味后的"人味"特征

### 6.1 好的写作特征

- [ ] 短句制造节奏（紧张时用短句，舒缓时用长句）
- [ ] 对话有个性（每个人说话方式不同）
- [ ] 动作代替心理（"他攥紧拳头"比"他很愤怒"好）
- [ ] 具体代替抽象（"三秒钟"比"片刻"好）

### 6.2 人味检查

| 特征 | 是否存在 |
|------|---------|
| 有口语化表达 | 是/否 |
| 有停顿/打断 | 是/否 |
| 有不完美的句子 | 是/否 |
| 有具体的细节 | 是/否 |

## 七、v1.1.1 新增原则

1. **替换而非删除**：禁用词必须有替换方案，不能直接删除
2. **语义完整性优先**：修改后必须语义通顺，不能产生歧义
3. **保留例外**：对话中可保留部分禁用词（角色说话可以有AI味，叙述不能）
4. **朗读测试**：替换后必须朗读测试，确认没有断裂

---

**记住：AI味是番茄平台的红线。每章必做去味检查，才能确保作品安全。**
