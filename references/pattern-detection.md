# 循环模式检测（v2.0.1新增）

> 基于大量实战审阅的经验总结
> 核心目标：检测并防止叙事模式无变化重复

---

## 一、实战教训

### 1.1 常见问题一览

| 模式 | 典型表现 | 重复次数 |
|------|---------|:---:|
| 遇新敌人→发现苦衷→对话解决→获力量 | 多个剧情段落高度相似 | **3次+** |
| 觉醒→升级→新敌人循环 | 全书后半段 | **4次+** |
| 高潮章名不副实 | 标题含"决战"但无战斗 | 多次 |

### 1.2 模式重复的危害

1. **审美疲劳**：读者发现套路，失去新鲜感
2. **可预测性**：读者能猜到接下来发生什么
3. **质量下滑**：模式化写作导致细节粗糙
4. **AI味加重**：模式重复是AI写作的典型特征

## 二、循环模式定义

### 2.1 核心循环模式

**模式A："觉醒-升级-新敌人"循环**

```
标准流程：
  1. 遇到新敌人/新挑战
  2. 发现现有力量不足
  3. 觉醒新能力/获得新节点
  4. 击败敌人
  5. 遇到更强的敌人
  6. 重复

问题：当这个循环重复3次以上且无变化时，读者会疲劳
```

**模式B："敌人有苦衷"循环**

```
标准流程：
  1. 遇到敌人
  2. 战斗/对峙
  3. 发现敌人有苦衷/被操控
  4. 通过对话/共鸣解决
  5. 敌人自我崩解/加入主角
  6. 重复

问题：这是最常见的模式重复问题之一
```

**模式C："探索新世界"循环**

```
标准流程：
  1. 进入新世界/新区域
  2. 发现新规则
  3. 快速通过（一笔带过）
  4. 到达目标
  5. 重复

问题：多个区域探索，部分一笔带过
```

### 2.2 章节结构模式

```
章节结构特征：
  - 开头类型：对话/描写/动作/回忆
  - 主体类型：探索/战斗/对话/回忆/升级
  - 结尾类型：悬念/解决/过渡/高潮
  - 冲突解决：战斗/对话/逃避/融合/话疗

若连续多章结构高度相似（相似度>70%），视为模式重复
```

## 三、循环模式检测方法

### 3.1 章节结构特征提取

```
对每章提取以下特征：

开头特征：
  - 第一段类型（对话/描写/动作）
  - 前100字关键词

主体特征：
  - 主要场景数量
  - 主要活动类型
  - 对话占比区间
  - 是否有战斗
  - 是否有升级/觉醒

结尾特征：
  - 最后一段类型
  - 是否有悬念
  - 冲突是否解决

解决方式：
  - 战斗解决
  - 对话解决
  - 话疗解决
  - 融合/共存
  - 逃避/推迟
```

### 3.2 相似度计算

```
对于相邻章节或同类型章节：

相似度 = 相同特征数 / 总特征数 * 100%

示例：
  章节A特征：[对话开头, 探索主体, 悬念结尾, 话疗解决]
  章节B特征：[对话开头, 探索主体, 悬念结尾, 话疗解决]
  相似度 = 4/4 = 100% → 🔴 完全重复
```

### 3.3 模式匹配

```
检测特定模式的出现次数：

模式 = "遇新敌人 → 发现苦衷 → 话疗解决"

for each 章节序列:
    if 匹配模式:
        模式计数 += 1
        记录出现位置

if 模式计数 >= 3:
    → 🔴 报警：模式重复3次+
```

## 四、循环模式检测流程

### 4.1 每5章检测

```
步骤1：提取最近5章结构特征
步骤2：计算章节间相似度
步骤3：检测是否有模式重复
步骤4：记录到 pattern-detection.md
步骤5：若发现问题，发出预警
```

### 4.2 每卷回顾

```
步骤1：统计本卷所有模式出现次数
步骤2：对比其他卷的模式分布
步骤3：检测是否有新模式出现
步骤4：评估模式多样性
```

### 4.3 全书审计

```
步骤1：绘制模式分布图
步骤2：检测是否有模式垄断
步骤3：评估模式创新程度
步骤4：提出改进建议
```

## 五、模式重复预警规则

| 触发条件 | 预警级别 | 说明 |
|---------|:---:|------|
| 同模式出现2次 | ⚠️ 提醒 | 需注意变化 |
| 同模式出现3次 | 🔴 报警 | 必须变化 |
| 连续3章结构相似度>70% | 🔴 报警 | 章节结构重复 |
| 卷内模式种类<3 | ⚠️ 提醒 | 模式多样性不足 |
| 话疗解决占比>30% | 🔴 报警 | 话疗模式泛滥 |

## 六、模式变化策略

### 6.1 打破循环的方法

**方法1：插入意外**

```
标准循环：遇敌→苦衷→话疗→解决
打破：遇敌→苦衷→话疗失败→必须战斗→代价惨重
```

**方法2：改变解决方式**

```
上次：话疗解决
这次：战斗解决
下次：智取解决
再下次：牺牲解决
```

**方法3：改变敌人类型**

```
上次：被操控的敌人（可话疗）
这次：无意识的怪物（必须战斗）
下次：有逻辑的智者（智取）
```

**方法4：改变代价**

```
上次：无损
这次：受伤
下次：牺牲队友
再下次：牺牲自己某部分
```

### 6.2 章节结构变化

```
若连续3章结构相似，第4章必须变化：

示例：
  连续3章：对话开头 + 探索主体 + 悬念结尾
  第4章：动作开头 + 战斗主体 + 解决结尾（强制变化）
```

## 八、高潮章专项检测

### 8.1 高潮章定义

```
高潮章特征（满足任一）：
  - 标题含"决战/最终/高潮/击杀"等关键词
  - outline.md标记为"高潮章"
  - 本章出现BOSS级敌人
  - 卷末章节
```

### 8.2 高潮章检测项

```
检测项：
  1. 是否有真正的战斗/冲突？
  2. 冲突是否通过战斗解决？
  3. 是否有代价/牺牲？
  4. 是否有反转/意外？
  5. 是否推进了主线？

反面教材（anti-pattern）：
  - 标题含"决战"但无战斗，话疗解决
  - 标题含"核心之战"但主要是内心独白
```

### 8.3 高潮章质量标准

| 检查项 | 标准 | 不合格案例 |
|-------|------|---------|
| 战斗描写 | ≥3次攻击动作 | 决战章：0次攻击 |
| 解决方式 | 战斗/智取 | 决战章：话疗解决 |
| 代价 | 有代价 | 最终章：主角无损 |
| 反转 | 有意外 | 多数无反转 |

---

**记住：模式重复是读者流失的主要原因之一。每重复一次，读者耐心就少一分。**
