### 核心问题回顾

长篇网文创作中反复出现的典型质量问题：

| 问题 | 表现 | 严重程度 |
|------|------|---------|
| "话疗"泛滥 | 大量章节含说服类关键词，多个BOSS被对话解决 | 致命 |
| 战斗描写崩溃 | 后期战斗场景几乎消失，意象重复率极高 | 致命 |
| 对话占比失控 | 对话占比随篇幅推进持续飙升 | 严重 |
| 节奏控制失败 | 篇幅超出故事承载力，后期注水严重 | 严重 |
| 循环模式重复 | "觉醒-升级-新敌人"等固定模式无变化重复 | 中等 |
| 反派降智 | 终极BOSS逻辑崩塌，轻易被说服 | 致命 |

## 核心定位

本 Skill 解决**八个**核心问题：

1. **长篇不跑偏**：通过 Truth Files 系统维护世界观、角色、伏笔、状态
2. **番茄能签约**：内置番茄平台规则、商业审查、付费转化检查
3. **AI没味道**：智能 AI 去味检查（替换而非删除），避免被平台限流
4. **会话连续性**：通过 handoff_current.md 实现跨会话无缝衔接
5. **规划能落地**：进度追踪面板，实时监控字数/章节偏离
6. **战斗有张力**：战斗场景专项审计，杜绝"话疗"解决BOSS
7. **对话不注水**：对话占比监控，防止全章变成聊天记录
8. **节奏不崩塌**：高潮密度追踪，防止后期注水和循环重复

## 最小审计集（v2.0.1 核心改进）

每章写完后**必须执行**以下8项检查，不可跳过：

### 检查1-5：继承自 v1.1.1

详见 v1.1.1 文档。

### 检查6：战斗场景检查（维度30，v2.0.1新增）

```
触发条件：
  - 章节标题含"战斗/决战/对抗/最终/击杀"等关键词
  - outline.md 标记为"高潮章"
  - 本章出现BOSS级敌人

检查项：
  1. 是否有实质性攻击/防御动作？（至少3次）
  2. 是否通过"说服/理解/感化"解决冲突？ -> 禁止！
  3. 战斗意象是否重复？
  4. 是否有战术博弈？（而非纯力量对轰）
  5. 是否有代价/牺牲？

评级标准：
  PASS 优秀：有战术、有代价、意象丰富
  WARN 及格：有战斗但模式化
  FAIL 不合格：无战斗或"话疗"解决
```

详见：`references/battle-scene-guide.md`

### 检查7：对话占比检查（维度31，v2.0.1新增）

```
计算方法：
  对话行数 = 统计引号内内容行数
  总行数 = 章节总行数
  对话占比 = 对话行数 / 总行数

标准：
  PASS 优秀：<= 25%（氛围/叙事为主）
  PASS 良好：26-35%（平衡得当）
  WARN 偏高：36-40%（需关注）
  FAIL 超标：> 40%（必须压缩对话）

趋势监控：
  - 记录每章对话占比
  - 若连续3章占比上升 -> 预警
  - 若占比 > 50% -> 立即报警
```

详见：`references/dialogue-monitor.md`

### 检查8：话疗模式检查（维度32，v2.0.1新增）

```
检测关键词：
  - "我理解你"
  - "说服"/"感化"/"唤醒"
  - "不是消灭，是救赎"
  - "融入"/"共存"/"和解"
  - BOSS被对话解决（三句话内放弃抵抗）

检查规则：
  1. 普通敌人：可通过对话解决（限1次/卷）
  2. 精英敌人：对话解决需铺垫3章+
  3. BOSS级敌人：禁止通过对话解决！必须战斗！
```

### 检查9：循环模式检查（维度33，每5章执行）

```
检测模式：
  1. "遇到新敌人 -> 发现苦衷 -> 对话解决 -> 获得力量"是否重复？
  2. 连续3个章节结构是否高度相似？
  3. 高潮章是否真的有高潮？

预警规则：
  - 同模式出现2次 -> WARN 提醒
  - 同模式出现3次+ -> FAIL 报警，要求变化
```

详见：`references/pattern-detection.md`

## 进度追踪面板（v2.0.1升级）

### 新增：节奏质量评估

```
高潮密度：
  高潮章数量 / 总章数
  标准：>= 15%（每7章至少1个高潮）

冲突强度趋势：
  统计每章冲突解决方式
  战斗解决占比应 >= 对话解决占比

节奏曲线：
  绘制章节质量曲线
  检测是否"前高后低"崩塌
```

详见：`assets/project-template/progress_tracker.md`

## 会话连续性工作流

### 新会话启动流程

```
1. 用户请求继续写作
2. AI请求加载 handoff_current.md
3. 用户提供文件内容（或AI从项目文件夹读取）
4. AI确认加载成功，报告当前进度
5. 继续写作或审计
```

### 会话交接协议

**AI启动时必须说**：
> "请提供 handoff_current.md 文件内容，我将加载当前进度。"

**加载成功后必须确认**：
> "已加载 handoff_current.md，当前进度：ch{{N}}，下一章：ch{{N+1}}。
> 主角当前在[地点]，持有[物品]，待处理伏笔[X个]。
> 总进度：{{actual}}字/{{target}}字（{{percent}}%）。
> 对话占比趋势：{{trend}}。上次高潮章：ch{{last_climax}}。"

## 教训库（v2.0.1）

> 以下是从大量长篇网文实战创作与深度审阅中总结的典型问题模式，必须在审计时特别关注。

### 问题1-8：继承自 v1.1.1

（略，见 v1.1.1 文档）

### 问题9：战斗描写崩溃（v2.0.1新增，严重）
- **表现**：中后期战斗场景逐渐消失，被大段对话取代
- **根因**：无战斗场景专项检查，主题表达压过情节张力
- **预防**：维度30强制检查，高潮章必须有实质性战斗描写

### 问题10：对话占比失控（v2.0.1新增，严重）
- **表现**：对话占比随篇幅推进持续飙升，后期章节近乎对话体
- **根因**：无对话占比监控，后期写作倾向用对话凑字数
- **预防**：维度31持续监控，上限40%，超标必须压缩

### 问题11："话疗"模式泛滥（v2.0.1新增，严重）
- **表现**：多个重要敌人被对话/说服解决，缺乏正当战斗
- **根因**：无"话疗"检测机制，主题表达滥用为万能解法
- **预防**：维度32检测，BOSS级敌人禁止被说服

### 问题12：循环模式重复（v2.0.1新增，中等）
- **表现**：固定叙事循环（如"觉醒-升级-新敌人"）无变化反复出现
- **根因**：无循环模式检测，写作陷入套路化
- **预防**：维度33检测，同模式出现3次以上强制报警

### 问题13：反派降智（v2.0.1新增，严重）
- **表现**：终极BOSS逻辑崩塌，轻易被主角说服或击败
- **根因**：急于表达主题，牺牲反派行为逻辑与角色尊严
- **预防**：BOSS必须有独立且自洽的逻辑体系，主角需找到逻辑漏洞而非简单"理解"

### 问题14：节奏崩塌（v2.0.1新增，严重）
- **表现**：作品前半部质量稳定，后半部出现灾难性下滑
- **根因**：无节奏质量评估机制，后期注水赶工
- **预防**：节奏曲线持续监控，"前高后低"趋势及时报警
