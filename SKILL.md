# 番茄小说写作Skill（Fanqie Novel Skill）

版本：**1.1.1**

**专为番茄小说平台打造的长篇网文写作操作系统。**

---

## v1.1.1 更新说明

本版本基于**280章、78万字实战创作**的经验教训，重点解决以下问题：

| 问题 | v1.1.0 | v1.1.1 |
|------|--------|--------|
| 规划执行脱节（120万字规划→78万字实际） | 无追踪机制 | 新增 progress_tracker.md 进度追踪面板 |
| AI去味破坏语义（机械删除导致读感下降） | 禁用词列表→搜索删除 | 智能替换表+语义完整性检查 |
| 情节跳脱无因果（场景A直接跳到场景C） | 维度28概念存在但未执行 | 新增维度29因果链检查+执行流程 |
| 时间黑洞（11点到→凌晨3点进，中间无交代） | 无检查 | 新增时间线填充检查 |
| 审计维度空转（28维度太复杂，AI跳过） | 列出28维度，无强制 | 最小审计集（5项强制，不执行=不通过） |

## 核心定位

本Skill解决五个核心问题：

1. **长篇不跑偏**：通过Truth Files系统维护世界观、角色、伏笔、状态
2. **番茄能签约**：内置番茄平台规则、商业审查、付费转化检查
3. **AI没味道**：智能AI去味检查（替换而非删除），避免被平台限流
4. **会话连续性**：通过handoff_current.md实现跨会话无缝衔接
5. **规划能落地**：进度追踪面板，实时监控字数/章节偏离

## 核心工作流

```
init → write-next → draft → ai-detox → audit → revise → extract-state → state-update → handoff-update
```

### v1.1.1 工作流强化

```
写前（强制3项）：
  ├─ 读取 current_state.md → 确认地点/时间/状态
  ├─ 读取 outline.md → 确认本章目标和字数
  └─ 读取 pending_hooks.md → 确认待处理伏笔

写后（强制5项——最小审计集）：
  ├─ 1. 字数检查（2000-2500字）
  ├─ 2. AI去味（智能替换，非机械删除）
  ├─ 3. 因果链检查（维度29，无跳脱）
  ├─ 4. 行为-规则一致性检查（维度28）
  └─ 5. 时间线检查（无黑洞）

状态更新（强制）：
  ├─ current_state.md
  ├─ pending_hooks.md
  ├─ handoff_current.md
  └─ progress_tracker.md（新增）
```

## 进度追踪面板（v1.1.1新增）

### 为什么需要

**实战教训**：规划120万字，实际只写了78万字（缩水35%）。原因是写作过程中没有进度追踪，outline.md被逐渐忽略，导致后期加速赶工、章节缩水。

详见：`assets/project-template/progress_tracker.md`

## 最小审计集（v1.1.1核心改进）

每章写完后**必须执行**以下5项检查，不可跳过：

### 检查1：字数检查
- 目标：2000-2500字
- 结果：✅通过 / ⚠️不足 / ⚠️超标

### 检查2：AI去味检查（智能替换）
- 搜索禁用词 → 分析语境 → 智能替换 → 语义验证
- 详见：`references/ai-detox-guide.md`

### 检查3：因果链检查（维度29，新增）
- 检查场景转换是否有因果连接
- 检查"突然""然后"是否虚假连接

### 检查4：行为-规则一致性检查（维度28）
- 检查主角行为是否与规则一致
- 规则怪谈类必查

### 检查5：时间线检查（新增）
- 检查时间跳跃是否有交代
- 检测时间黑洞（跳跃>1小时无交代）

详见：`references/audit-dimensions.md`

## 会话连续性工作流

### 新会话启动流程

```
1. 用户请求继续写作
2. AI请求加载 handoff_current.md
3. 用户提供文件内容（或AI从项目文件夹读取）
4. AI确认加载成功，报告当前进度
5. 继续写作或审计
```

### 会话交接协议

**AI启动时必须说**：
> "请提供 handoff_current.md 文件内容，我将加载当前进度。"

**加载成功后必须确认**：
> "已加载 handoff_current.md，当前进度：ch{{N}}，下一章：ch{{N+1}}。
> 主角当前在[地点]，持有[物品]，待处理伏笔[X个]。
> 总进度：{{actual}}字/{{target}}字（{{percent}}%）。"

## 历史问题总结（教训库）

> 这些是从**280章实战创作**中发现的问题，必须在审计时特别关注。

### 问题1：地点矛盾
- 根因：写作时未查阅current_state.md
- 预防：每次写作前必须确认地点

### 问题2：设定不一致
- 根因：设定未在story_bible.md中明确定义
- 预防：所有核心设定必须明确定义

### 问题3：行为-规则矛盾（最严重）
- 根因：维度28未执行
- 预防：规则怪谈类必须逐章执行维度28检查

### 问题4：AI去味破坏语义（v1.1.1重点修复）
- 根因：机械删除禁用词，未考虑语义完整性
- 预防：智能替换，语义完整性优先

### 问题5：章节字数超规
- 根因：未按outline.md目标写作
- 预防：写作前确认字数限制

### 问题6：规划执行脱节（v1.1.1新增，最严重）
- 根因：无进度追踪机制，outline.md被忽略
- 预防：每章更新progress_tracker.md

### 问题7：情节跳脱无因果（v1.1.1新增）
- 根因：无因果链检查，时间黑洞未被发现
- 预防：每章执行维度29检查

### 问题8：审计维度空转（v1.1.1新增）
- 根因：维度太多、无优先级、无强制执行机制
- 预防：最小审计集强制执行
